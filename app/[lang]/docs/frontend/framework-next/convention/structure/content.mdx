# 프로젝트 파일 및 폴더 구조

이 페이지는 파일 및 폴더 구조를 프로젝트에 맞게 설계할 수 있는 개요를 제공합니다.
해당 문서는 아래 문서들의 개념을 이미 읽었다고 가정합니다.

- [NextJs Project Structure](https://nextjs.org/docs/getting-started/project-structure)
- [App router Routing](https://nextjs.org/docs/app/building-your-application/routing)

### 키워드

- [under score folder](https://nextjs.org/docs/app/building-your-application/routing/colocation#private-folders): underscore prefix(예시 \_components) 인 폴더는 라우팅 목적이 아닌 모듈에 대한 네이밍으로 라우터에서 제외됩니다.
- [parenthesis folder](https://nextjs.org/docs/app/building-your-application/routing/colocation#route-groups): parenthesis prefix(예시 auth) 인 폴더는 로직 그룹화를 위한 네이밍으로 라우터에서 제외됩니다.

### 모듈 구조

또한, 폴더 내부의 파일은 해당 폴더의 기능을 설명하는 파일이어야 합니다.
